# Agent Marketplace — User Onboarding Brief

## Overview

This brief outlines how new users discover, evaluate, purchase, and successfully use AI agent configuration files from our marketplace. The goal is a frictionless journey from curiosity to working agent—in under 10 minutes.

---

## User Journey

### Step 1: Landing

Users arrive via Reddit, Twitter/X, or word of mouth. The landing page makes one thing clear: **"Copy-paste ready agent configs that work."**

**What they see:**
- Hero section with a single value proposition and a "Download Free Starter Pack" button
- Three featured products with icons, prices, and ratings
- Platform tags prominently displayed: "Works with OpenClaw, Claude, GPTs"

**Friction here:** None. No email required for the free pack. One click, one download.

### Step 2: Browse

Users explore via category grid or search. Categories include Starter Packs, Persona Files, Workflow Templates, Skills, and Industry-Specific configs.

**Key features:**
- Filter by platform (where the config will run)
- Filter by price range
- Sort by rating, popularity, or newest

**What browsing feels like:** Skimmable cards with clear names, one-line descriptions, and social proof (stars, download count). No wall of text.

### Step 3: Preview

Every product page shows the first 50% of the actual config file. Users read the SOUL.md or AGENTS.md before buying—they see exactly what they're getting.

**What preview includes:**
- File structure overview (what files are included)
- Key capability highlights
- Dependencies or requirements
- Platform compatibility notes

**Why this matters:** Users trust what they can see. No mystery. No surprises post-purchase.

### Step 4: Purchase

Checkout is minimal: email for receipt, Stripe payment, done. No account creation required.

**Checkout flow:**
1. Add to cart (or direct buy)
2. Email + payment
3. Instant download link appears
4. Receipt sent to email

**Time to complete:** Under 2 minutes.

### Step 5: Download

Users get a .zip containing all config files plus a README with installation instructions.

**What's in the download:**
- SOUL.md, AGENTS.md, skills/ folder as applicable
- Platform-specific variants (e.g., OpenClaw vs. Claude versions)
- README.md with step-by-step placement guide
- Example command: `cp research-agent-config/* ./your-agent/`

### Step 6: Install

This is where most marketplaces lose users. We solve it with:

- **Explicit file paths:** "Place SOUL.md in /config/" not "put it somewhere appropriate"
- **Platform-specific instructions:** Separate steps for OpenClaw, Claude, GPTs, LangChain
- **Quick test:** One-liner to verify the config loaded correctly

**Expected time:** 3–5 minutes from download to working agent.

### Step 7: Use

The config is now active. Users see immediate results:

- Research Agent: starts searching, synthesizing, citing
- Writer Agent: begins analyzing tone, structuring content
- Persona File: agent adopts new tone and behavior

**Success signal:** The user experiences the value within minutes—not hours of configuration.

## Language and Tone

### Speak User, Not Engineer

**Avoid:** "Reactive agent architecture," "tool-chain pipeline integration," "context window optimization"

**Use:** "Agent that remembers your conversation," "connects to search and files," "handles longer conversations"

### Copy-Paste Ready Messaging

Every touchpoint reinforces: **"Download → Copy → It works."**

- Product descriptions: "Drop these files into your project and the agent is ready."
- README files: Step-by-step, platform-specific, test commands included
- Support responses: "Copy this command" not "configure your environment"

### Value-First, Not Salesy

- Lead with what the config does, not why they should buy
- Let preview content sell itself
- Social proof: "Used by 50+ teams" not "Best-selling config!"
- Bundles show savings transparently
