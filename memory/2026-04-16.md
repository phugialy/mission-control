## Buster Discord Mention Issue

**Problem:** Buster wasn' t responding to Discord mentions (@Buster).

**Investigation:**
- Tested Buster via subagent spawn → worked fine (responded "Buster online! 👋")
- Checked OpenClaw routing config - buster IS bound to Discord
- Sent direct message to Buster → delivered but no response

**Root Cause (Architecture):**
- OpenClaw architecture: main agent (Kai/mission-control) receives ALL messages in a channel
- Sub-agents (rex, pete, quill, scribe, buster) are spawned by the main agent
- They do NOT independently monitor Discord channels or respond to mentions
- This is by design, not a bug

**How to use Buster:**
- Tag @Kai- Lord and ask to spawn Buster for a task
- Or Kai spawns Buster when needed

**Options for the future:**
- Could configure auto-spawn on @mention (more complex)
- Current flow (ask Kai → spawn Buster) is safer and intentional
